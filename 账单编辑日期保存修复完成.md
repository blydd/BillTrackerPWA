# 账单编辑日期保存修复完成

## 问题描述
编辑账单时修改日期无法保存，保存后日期仍然是原来的值。

## 问题原因
在 `Repository/SQLiteRepository.swift` 的 `updateBill` 方法中，SQL 更新语句缺少了 `created_at` 字段的更新。

原来的 SQL 语句：
```sql
UPDATE bills SET amount = ?, payment_method_id = ?, owner_id = ?, note = ?, updated_at = ?
WHERE id = ?;
```

## 修复方案
修改 `updateBill` 方法的 SQL 语句，添加 `created_at` 字段的更新：

```sql
UPDATE bills SET amount = ?, payment_method_id = ?, owner_id = ?, note = ?, created_at = ?, updated_at = ?
WHERE id = ?;
```

同时调整了参数绑定的顺序，确保 `created_at` 字段能够正确更新。

## 修复内容
- 修改了 `Repository/SQLiteRepository.swift` 中的 `updateBill` 方法
- 在 SQL 更新语句中添加了 `created_at` 字段
- 调整了参数绑定顺序，确保所有字段都能正确更新

## 测试验证
修复后，编辑账单时修改日期应该能够正常保存。

## 影响范围
- 仅影响账单编辑功能
- 不影响账单创建功能
- 不影响其他数据的更新
