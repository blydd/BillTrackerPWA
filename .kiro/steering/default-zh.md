## 核心规则（必遵守）
### 1. 语言规则
- 所有回答默认使用**简体中文**，除非用户明确指定其他语言（如“用英文回答”“English”“日语で”等）。
- 技术术语保留英文时需补充中文解释（例如：“API（应用程序编程接口）”）。
- 代码注释、错误提示、文档说明均使用中文；代码本身（变量名、函数名）可保持英文。
- 若用户输入为非中文且未指定语言，默认用中文解释内容（例如用户输入“What is Kiro?”，回答中文释义）。

### 2. Git 操作规则
- 严禁自动生成/执行任何 Git 提交（git commit）、推送（git push）、合并（git merge）、拉取（git pull）等代码仓操作指令。
- 仅当用户**明确说明**需要执行 Git 提交/推送（如“帮我写 git 提交代码的命令”“生成 git push 指令”）时，才可提供对应的指令示例（需标注“仅示例，需用户手动执行”）。
- 提供 Git 指令时，必须附带中文风险提示（例如：“注意：执行 git push 前请确认代码已测试，避免推送错误代码至远程仓库”）。
- 禁止自动触发任何 Git 操作的脚本/自动化流程，所有 Git 相关操作需用户手动确认后执行。

### 3. 报错处理
- 当执行命令报错时，不要自动终止，把要修改的内容详细输出让用户手动复制到指定文件。
